# 100DaysOfCode 全体方針

## 目的

今回の100DaysOfCodeでは、主に以下を目的とする。

* 技術力向上
* 継続習慣を作る
* 新しい技術を試す

ポートフォリオ化やキャリア成果物としての完成度は必須にしない。
まずは毎日手を動かし、言語仕様・実装力・設計力を伸ばす。

---

## 基本ルール

* 毎日1時間コードを書く
* 毎日 `#100DaysOfCode` 付きで投稿する
* GitHubにも作業記録を残す
* 1日休んだ場合は最後に1日追加する
* 2日連続では休まない
* 交流ルールは努力目標とする

---

## 伸ばしたい力

### 必ず伸ばしたい力

* 言語仕様の理解
* 実装力
* 設計力

### 追加で伸ばしたい力

* UI実装力
* DB設計・データ処理
* Windowsアプリ開発

ただし、すべてを1つのアプリに詰め込まない。
作るものごとに伸ばす力を分ける。

例：

* Rust CLIで言語仕様・ファイル処理を伸ばす
* TauriでRust + TypeScript + UI実装を伸ばす
* C++でDB設計やWindowsツール開発に集中する

## 成果物の方針

100日で1つの大きなアプリを完璧に作るより、**小さいアプリを複数作る**方向を有力とする。

それぞれのアプリで伸ばす技術を分ける。

例：

* 1本目：Rust CLI
* 2本目：Tauri GUIアプリ
* 3本目：C++ + SQLiteツール
* 4本目：TypeScriptデータ可視化

実際に何本作るかは途中で判断する。

## 候補選定の観点

今後、次に作るものを選ぶときは以下で判断する。

* 技術力が伸びそうか
* 面白そうか
* 100日続けられそうか
* 小さく完成できそうか
* 今の力量に合っているか
* スコープが広がりすぎないか
* 使う言語と伸ばしたい力が合っているか

---

## 方針

100DaysOfCode全体では、完璧な計画を最初から作らない。
まず1本目を小さく始め、作りながら次に取り組むものを決めていく。
